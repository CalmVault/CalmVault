# Contributing to CalmVault 🛡️

Thank you for your interest in contributing to **CalmVault** – a privacy-first, decentralized therapy platform on the Aztec Network. Before making any changes, we encourage you to discuss your ideas via an issue, email, or community channel to ensure alignment with our privacy-centric vision.

## 💬 Connect With Us

Have questions or ideas? Join our [community discussions](https://t.me/) to collaborate with other privacy enthusiasts and developers.

## 🛠 Development Environment Setup

To get started with local development, follow these steps:

```bash
git clone https://github.com/CalmVault/CalmVault.git
cd calmvault
npm install
npm run dev
```

Make sure you have:

* Node.js
* npm or yarn
* Hardhat (for smart contract development)
* A zk-money wallet (for testing shielded payments)

## 🚀 Issues and Feature Requests

Look for issues labeled **open for contribution** — these are specifically available to the community.

Please pay attention to any deadlines or priority notes mentioned in the issue descriptions. Delays might lead to reassignments or impact any reward eligibility.

## 📌 Creating an Issue

Found a bug or have a feature request? Submit an issue!

Before submitting:

* **Search existing issues** to avoid duplicates.
* Make your report:

  * **Reproducible**: Provide clear steps.
  * **Specific**: Include details like environment, logs, screenshots.
  * **Unique**: One issue per topic.
  * **Scoped**: Focus on a single improvement or bug.

Even better—submit a **pull request** with your proposed fix!

## 🔀 How to Submit a Pull Request

1. **Check for existing PRs or issues** related to your change.
2. **Fork the repo** and create your branch:

   ```bash
   git checkout -b feat-anonymous-signup
   ```
3. **Commit your changes** with a clear message:

   ```bash
   git commit -m 'feat: add anonymous signup via zk-proofs'
   ```
4. **Push your branch** to your fork:

   ```bash
   git push origin feat-123-anonymous-signup
   ```
5. **Open a Pull Request** on GitHub.

Please link related issues in your PR description.

## 🏷 Pull Request Labels

We use labels to manage PRs effectively:

* **Ready for Review**: Completed and awaiting feedback.
* **In Progress – Do Not Merge**: Still being developed.
* **Changes Requested**: Modifications needed from reviewers.
* **Security Critical**: Involves privacy or encryption features; reviewed carefully.

## 🧾 Code of Conduct

All contributors are expected to uphold our community values:

* Respect anonymity and privacy
* Be kind and constructive
* Report security issues responsibly
